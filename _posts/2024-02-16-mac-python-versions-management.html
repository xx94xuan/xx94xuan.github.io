Different tools to manage Python versions

venv, pyenv, virtualenv and pipenv are all tools used for managing Python environments, but they serve different purposes and have some distinct features. Here's a brief overview of each:
    venv:
  • module in Python 3 that allows you to create lightweight virtual environments.
  • provides a way to create isolated Python environments with their own Python interpreter and installed packages.
  • recommended for creating Python 3 virtual environments.
    pyenv:
  • third-party tool used for managing multiple Python versions on your system.
  • allows you to easily switch between different Python versions and set the global Python version or project-specific Python versions.
  • useful when you need to work with multiple Python versions simultaneously.
    3. virtualenv:
  • third-party package that allows you to create isolated Python environments.
  • works with both Python 2 and Python 3 and is compatible with older versions of Python.
  • provides more advanced features compared to venv, such as the ability to clone an existing environment or create environments with specific Python versions.
    4. pipenv:
  • higher-level tool that combines pip package management and virtualenv functionality.
  • aims to simplify dependency management and provide an easier workflow for Python projects.
  • automatically creates and manages virtual environments and tracks package dependencies in a Pipfile and Pipfile.lock file.
  • integrates package installation, dependency resolution, and environment creation into a single tool.

Use `pyenv` to manage Python versions on Mac

  • install pyenv and its dependencies:
    • brew install pyenv
  • list all of the availble Python versions you can install:
    • pyenv install --list
  • to install specific version:
    • pyenv install [VERSION]
  • Set the global Python version:
    • pyenv global [VERSION]
  • Set an application-specific Python version:
    • pyenv local [VERSION]
    This command creates a .python-version file in your current directory. If you have pyenv active in your environment, this file will automatically activate this version for you.
  • Create a virtual enviroment:
    • brew install pyenv-virtualenv
    pyenv virtualenv [VERSION] [ENV_NAME]
    This will create a new virtual environment in ~/.pyenv/versions/[ENV_NAME].
  • Activate the virtual enviroment:
    • pyenv activate [ENV_NAME]
    pyenv deactivate
    This will activate/deactivate the virtual env and make it the default Python environment for your terminal session.
